## 摘要
斜视是一种常见的眼部疾病，需要精细的亚型诊断以制定个体化治疗方案。然而，现有的深度学习方法主要提供诊断预测，缺乏透明的推理过程，而近期的大视觉语言模型（LVLMs）尽管在联合图像理解和报告生成方面具有前景，但在这种证据敏感且规则驱动的医学任务中仍然极易产生幻觉。为应对这些挑战，我们提出了MAGIS，一个基于证据的多智能体推理框架，用于可解释的斜视诊断。MAGIS将黑盒端到端生成转化为一个结构化的诊断过程，包括候选假设生成、双证据约束上下文、基于证据的校正验证和报告生成。具体而言，我们引入了一种双证据约束上下文（DECC）机制，该机制将来自九方位眼位照片的视觉证据和基于证据的临床诊断规则联合组织成一个约束上下文，用于可靠的诊断推理。我们进一步开发了一种基于证据的校正验证（EBCV）机制，该机制验证当前诊断假设是否得到视觉证据、基于热图的视觉线索以及基于证据的临床诊断规则的支持。当检测到不一致时，会触发假设修正。在细粒度斜视基准上的实验表明，MAGIS不仅显著优于其他最先进的诊断系统，将加权F1分数从72.0%提高到91.3%，而且大幅提高了生成的诊断报告的临床可靠性（一致性、对齐度和完整性）。这些结果表明，MAGIS为构建准确、基于证据且临床可解释的斜视诊断系统提供了一种有效的解决方案。

## Abstract
Strabismus is a common ocular disorder that requires fine-grained subtype diagnosis for individualized treatment planning. However, existing deep learning methods mainly provide diagnostic predictions without transparent reasoning, while recent large vision-language models (LVLMs), although promising for joint image understanding and report generation, remain highly prone to hallucination in this evidence-sensitive and rule-driven medical task. To address these challenges, we propose MAGIS, an evidence-based Multi-AGent reasoning for Interpretable Strabismus diagnosis framework. MAGIS transforms black-box end-to-end generation into a structured diagnostic process consisting of candidate hypothesis generation, dual-evidence constrained context, evidence-based corrective verification, and report generation. Specifically, we introduce a Dual-Evidence Constrained Context (DECC) mechanism that jointly organizes visual evidence from the photograph of the nine cardinal positions of gaze and evidence-based clinical diagnostic rules into a constrained context for reliable diagnostic reasoning. We further develop an Evidence-Based Corrective Verification (EBCV) mechanism that verifies whether the current diagnostic hypothesis is supported by visual evidence, heatmap-based visual cues, and evidence-based clinical diagnostic rules. Hypothesis refinement is triggered when inconsistency is detected. Experiments on a fine-grained strabismus benchmark demonstrate that MAGIS not only significantly outperforms other state-of-the-art diagnostic systems, improving the weighted F1 score from 72.0% to 91.3%, but also substantially improves the clinical reliability (consistency, alignment, and completeness) of generated diagnostic reports. These results demonstrate that MAGIS provides an effective solution for building accurate, evidence-based, and clinically interpretable strabismus diagnosis systems.

## 2. 论文提出的方法论：核心思想、关键技术细节
### 核心思想
- 将诊断分解为**候选假设生成 → 双证据约束上下文构建 → 基于证据的校正验证 → 报告生成**四个阶段，通过多智能体协作实现证据驱动的闭环验证。
- 关键在于**DECC**（双证据约束上下文）和**EBCV**（基于证据的校正验证）两个机制：DECC联合视觉证据与临床诊断规则形成约束上下文；EBCV显式检查假设是否被证据支持，若不支持则触发修正。

### 关键技术细节
#### (a) 多智能体架构
- **Classifier Agent**：基于Complementary Decoupled Attention (CDA)模块，从九方位眼位图中提取**帧内注意力热图**（局部对齐线索）和**帧间注意力热图**（跨眼位变异线索），输出置信度排序的候选亚型列表 \(\{ (k_i(\mathbf{X}), p_i(\mathbf{X})) \}_{i=1}^n\) 及热图 \(\mathbf{H}_{\text{intra}}, \mathbf{H}_{\text{inter}}\)。
- **Verifier Agent**：接收DECC构建的上下文，执行EBCV。若当前最佳候选通过验证，则送Generator；否则选择下一候选重新验证。
- **Generator Agent**：将验证通过的预测组织为结构化报告，包含关键异常、模式与角度分析、校正验证结果、治疗建议及所用规则。

#### (b) Dual-Evidence Constrained Context (DECC)
- **视觉证据**：由CDA模块产生的输入图像 \(\mathbf{X}\)、帧内热图 \(\mathbf{H}_{\text{intra}}\)、帧间热图 \(\mathbf{H}_{\text{inter}}\) 组成：\(\mathbf{X}_{\text{visual}} = \{ \mathbf{X}, \mathbf{H}_{\text{intra}}, \mathbf{H}_{\text{inter}} \}\)。
- **规则证据**：通过**医生在环（Doctor-in-the-Loop）过程**将斜视指南形式化为三级规则集：通用规则 \(\mathcal{R}_G\)、斜视规则 \(\mathcal{R}_S\)、亚型规则 \(\mathcal{R}_{\text{sub}}\)。对候选预测 \(k_i(\mathbf{X})\)，检索对应规则 \(\mathcal{R}_i(\mathbf{X})\)。
- **组合**：\(\mathbf{C}_{\text{DECC}} = \{ \mathbf{X}_{\text{visual}}, \mathcal{R}_i(\mathbf{X}) \}\)。

#### (c) Evidence-Based Corrective Verification (EBCV)
- **规则引导验证（Rule-guided CV）**：检查当前亚型规则定义的预期异常是否与视觉证据匹配。使用LVLM描述图像异常并与规则对照。
- **热图引导验证（Heatmap-guided CV）**：检查帧内/帧间热图关注的区域是否与视觉证据中的异常区域一致。要求至少一个热图通过。
- **校正精化**：若当前候选未通过双重验证，则拒绝该假设，选择置信度列表中下一候选重新构建DECC并验证。迭代直至有候选通过或列表耗尽。
- **最终验证**：规则引导CV为必要条件，热图引导CV为辅助条件（至少一个热图一致）。

#### (d) 报告生成
- 生成包含**关键异常、模式与角度分析、校正验证结果、治疗建议、所用规则**的结构化报告，每个部分均源自前序验证结果，而非自由生成。

## 7. 优点
1. **创新方法论**：将诊断建模为显式的证据验证闭环，而非单纯依赖语言模型的能力，从根本上减少幻觉。

6. **灵活性**：框架模块化（视觉证据模块、规则集可替换），易于扩展至其他医学应用。
