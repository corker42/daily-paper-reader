## 摘要
糖尿病视网膜病变（DR）是可预防性失明的主要原因之一，但农村地区往往缺乏早期检测所需的专家和基础设施。尽管基于云的深度学习系统具有高精度，但由于高延迟、有限带宽和高数据传输成本，它们在这些环境中面临重大挑战。为了解决这些问题，我们在公共APTOS 2019失明检测数据集上提出了一种双层边缘-云级联架构。第1层在本地诊所设备上运行轻量级MobileNetV3-small模型，对可转诊DR（2-4级）和不可转诊DR（0-1级）进行二分类分诊。第2层在云端运行RETFoundDINOv2模型进行序数严重程度分级，但仅对第1层标记为可转诊的图像子集进行处理。在包含733张图像的分层APTOS测试集上，第1层在验证调优的高敏感度阈值下达到98.99%的灵敏度和84.37%的特异度。默认级联将49.52%的测试图像转发至第2层，相比对所有图像使用基于云的模型，减少了50.48%的云端调用。在部署的四类输出空间（0-1级/2级/3级/4级）中，级联架构获得80.49%的准确率和0.8167的二次加权kappa值；仅用云端的基线获得80.76%的准确率和0.8184的二次加权kappa值。在APTOS上，级联架构将云端使用减少约一半，而分级性能略有下降。索引术语：糖尿病视网膜病变，边缘-云级联，MobileNetV3-small，RETFound-DINOv2，视网膜筛查，远程眼科

## Abstract
Diabetic Retinopathy (DR) is one of the leading causes of preventable blindness, yet rural regions often lack the specialists and infrastructure needed for early detection. Although cloud-based deep learning systems offer high accuracy, they face significant challenges in these settings due to high latency, limited bandwidth, and high data transmission costs. To address these challenges, we propose a two-tier edge-cloud cascade on the public APTOS 2019 Blindness Detection dataset. Tier 1 runs a lightweight MobileNetV3-small model on a local clinic device to perform a binary triage between Referable DR (Classes 2-4) and Non-referable DR (Classes 0-1). Tier 2 runs a RETFoundDINOv2 model in the cloud for ordinal severity grading, but only on the subset of images flagged as referable by Tier 1. On a stratified APTOS test split of 733 images, Tier 1 reaches 98.99% sensitivity and 84.37% specificity at a validation-tuned high-sensitivity threshold. The default cascade forwards 49.52% of test images to Tier 2, reducing cloud calls by 50.48% relative to using a cloud-based model for all images. In the deployed 4-class output space (Class 0-1 / Class 2 / Class 3 / Class 4), the cascade obtains 80.49% accuracy and 0.8167 quadratic weighted kappa; the cloud-only baseline obtains 80.76% accuracy and 0.8184 quadratic weighted kappa. On APTOS, the cascade cuts cloud use by about half with a modest drop in grading performance. Index Terms: Diabetic Retinopathy, Edge-Cloud Cascade, MobileNetV3-small, RETFound-DINOv2, Retinal Screening, tele-ophthalmology

### 1. 论文的核心问题与整体含义（研究动机和背景）
糖尿病视网膜病变（DR）是导致可预防性失明的主要原因之一，但农村地区普遍缺乏眼科专家和诊断基础设施。虽然纯云端深度学习模型在 DR 检测上精度很高，但在农村场景下面临三大障碍：**高延迟**（血图像通常 5–20 MB，上传耗时长）、**有限带宽**（低带宽链接影响实时性）、**高传输成本**（数据按量计费增加运营成本）。此外，云模型依赖稳定网络连接，实际部署中曾因网络问题导致无法返回结果。因此，论文提出一个**两级边缘‑云级联架构**：将简单分诊任务放在本地轻量模型上执行，仅将疑似病例上传至云端进行精细分级，以在**保持较高诊断性能的同时大幅减少云端调用**，从而适应农村远程医疗场景。

### 2. 方法论：核心思想、关键技术细节
#### 2.1 核心思想
利用任务难度不对称性：**可转诊/不可转诊的二分类** 属于粗粒度分类，可由轻量模型在本地完成；而**序数严重程度分级**（ICDR 0–4 级）需要更强大的模型，放在云端处理。仅当本地模型预测为「可转诊」时才触发云端推理。

#### 2.2 两级管道
- **Tier 1（边缘分诊模型）**：使用 **MobileNetV3‑small**（基于 ClementP 发布的 FundusDRGrading 预训练权重），输出一个序数 logit，经 sigmoid 转换为可转诊得分。从验证集上选择高灵敏度阈值（σ(z) ≥ 0.6977），以减少假阴性（漏诊可转诊病例）。输入图像预处理：自动裁剪红色通道暗边框 → 保持长宽比 resize → padding 至 512×512 → ImageNet 归一化。
- **Tier 2（云端严重程度模型）**：使用 **RETFound‑DINOv2 ViT‑L/14** 作为骨干网络，附加单 logit 序数回归头（1024→1）。部分微调：只训练最后两个 transformer 块、post‑pool LayerNorm 和新头，其余冻结（约 25.2M 可训练参数）。输出连续得分，圆整并裁剪到 0–4 整数级。输入预处理：裁剪视网膜区域 → 缩放到 392×392 → ImageNet 归一化。
- **级联决策**：Tier 1 输出得分低于阈值的图像直接归为「不可转诊」（Class 0–1）；高于阈值的图像转发至 Tier 2，绕过云端处理留在本地；最终输出折叠为 4 个部署类：Class 0–1、Class 2、Class 3、Class 4。允许 Tier 2 将 Tier 1 误报的可转诊图像重新归为不可转诊（覆盖机制）。

#### 2.3 损失函数与优化
- Tier 1：使用预训练权重，未重新训练。
- Tier 2：**MSE 损失** 优化整数 ICDR 等级（序数回归），AdamW 优化，25 epoch，双精度混合精度，backbone 学习率 1e‑5，head 学习率 1e‑3，weight decay 0.01。基于验证集 5 类二次加权 kappa 选择最佳 checkpoint。

### 7. 优点
- **动机清晰且实用**：针对农村医疗的痛点（延迟、带宽、成本）设计，具有实际部署价值。

- **性能损失微小**：在减少一半云端调用的同时，分级准确率和 kappa 几乎持平，证明级联有效。
